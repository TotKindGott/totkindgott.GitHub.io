COLLECTION

Why build this website in the first place?

I needed a way to quickly and easily find out whether a specific model was already in the collection.

Everything is being tracked in a spreadsheet in the Numbers app.

Search and media attachment there is limited so a new approach was needed.

Long story short, now it is possible to find the answer in seconds by launching a shortcut and providing a search query.

How do I search your collection with the help of Shortcuts app?

Setup in Shortcuts is very simple. The shortcut consists of three steps.

The shortcut prompts for input, then encodes it and adds it to URL.

The webpage receives your search query as a URL argument and parses it, loads the content and filters through it.

How do I use this site?

on the top of the page helps find either relevant content on the page or collection-wide models.

Examples of page-specific search functions are: statistics for inventory and for collection completions.

Most other pages redirect search query as parameter to search.html.

What else is here besides Hot Wheels?

There is a reference page for treasure hunts, but of course that is also Hot Wheels.

All the info and images on that page are from HWTreasure.com.

How is this site made?

Everything is being generated from a comma-separated value file and a set of images.

From Numbers app .csv file is fed to Pythonista app, where all the data is crunched, and Photos app runs a shortcut on selected photos which resizes them and places them into a directory in Files app. Images get renamed there and also imported to Pythonista.

HTML gets generated and copied to Working Copy app, then it gets a commit to GitHub and within minutes an updated version is published. More info here.
